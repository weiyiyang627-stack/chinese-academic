# 信息架构与界面设计

## 1. 设计原则

- 默认语言为简体中文
- 原文英文只在必要时出现
- 以科研任务组织页面，不以模型能力组织页面
- 所有主要页面都支持中英切换和学术术语解释

## 2. 顶层信息架构

```text
首页
├─ 今日论文
├─ 我的主题
├─ 深度翻译
├─ 科研工作流
└─ 术语与变量库
```

右上角全局控件：

- 语言切换：中文 / English
- 术语偏好
- 我的项目
- 收藏夹
- 账号

## 3. 首页结构

### 3.1 首页区块

1. 今日推荐论文
2. 与你研究最相关的主题
3. 最近保存的术语与变量
4. 快捷工作流入口
5. 最近阅读与继续写作

### 3.2 视觉策略

- 首页主标题用中文，强调研究任务，例如“今天值得读的论文”
- 每张论文卡片先给中文标题，再给英文副标题
- 推荐理由使用中文自然解释，不堆砌标签

## 4. 每日论文卡片设计

```text
--------------------------------------------------
中文标题
英文标题
作者 | 期刊 | 发表状态

中文摘要（重写版）

为什么与你研究相关

推荐用途：理论 / 变量 / 方法 / 讨论 / inspiration

[只看中文总结] [一键翻译全文] [中英双栏] [原文链接]
--------------------------------------------------
```

## 5. 论文详情页

### 5.1 页面布局

支持两种模式：

- 标准模式：左导航 + 中内容 + 右侧辅助面板
- 对照模式：左中文 + 右英文

### 5.2 左侧导航

- 标题与元数据
- 中文摘要
- 研究问题
- 理论基础
- 方法
- 核心发现
- 理论贡献
- 关键变量
- 术语表
- 可引用句

### 5.3 固定工具栏

- 一键翻译全文
- 只看中文总结
- 对照阅读
- 导出综述笔记
- 加入变量库
- 生成写作素材

### 5.4 细节交互

#### hover 术语

显示：

- 英文原词
- 中文统一译法
- 简要定义
- 本文语境

#### 点击变量

弹出变量卡：

- 中文变量名
- 英文变量名
- 理论定义
- 测量方式
- 替代变量
- 可用于论文写作的表达方式

#### 选中文本

快捷动作：

- 加入我的综述
- 生成中文引用句
- 生成英文投稿表达

## 6. 科研工作流中心

工作流首页用中文任务卡组织：

- 文献综述生成
- 理论缺口识别
- 变量构建
- 结果解释
- 中文构思转英文投稿

每个工作流卡片都说明：

- 输入需要什么
- 输出会是什么
- 适合什么研究阶段

## 7. 术语与变量库

### 7.1 术语条目

每个术语包含：

- 英文术语
- 中文标准译法
- 用户偏好译法
- 备选译法
- 学科语境说明
- 示例论文

### 7.2 变量条目

每个变量包含：

- 中英文名称
- 理论定义
- 常见量表
- 替代指标
- 相关变量
- 可复用写作句式

## 8. 中文原生体验要求

产品文案应尽量使用中国学术用户熟悉的表达：

- 用“理论贡献提炼”代替“Summarize contribution”
- 用“变量命名建议”代替“Generate construct labels”
- 用“中文摘要重写”代替“Abstract translation”

界面不是把英文标签翻成中文，而是从中文科研习惯出发重构入口与流程。
