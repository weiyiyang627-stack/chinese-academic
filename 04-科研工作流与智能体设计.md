# 科研工作流与智能体设计

## 1. 总体思路

系统不应只提供单次问答，而应围绕真实科研任务构建一组可组合工作流。每个工作流都接收结构化输入，并输出符合中文学术风格的结果。

## 2. 工作流列表

### 2.1 文献综述工作流

输入：

- 主题
- 已选论文
- 目标章节用途

输出：

- 中文文献综述段落
- 理论流派梳理
- 研究分歧总结
- 理论缺口
- 英文综述草稿

适用阶段：

- 开题
- 综述写作
- 引言铺垫

### 2.2 变量构建工作流

输入：

- 研究现象描述
- 参考论文
- 目标期刊风格

输出：

- 中文变量命名
- 英文变量命名
- 构念定义
- 推荐测量方式
- 替代变量与代理指标
- 与相近变量的区分

### 2.3 实证分析工作流

输入：

- 假设信息
- 回归或 SEM 结果
- 相关变量定义

输出：

- 中文结果解释
- 英文论文表达
- 与理论的一致或不一致说明
- 结果讨论要点

### 2.4 写作工作流

输入：

- 中文构思
- 参考文献
- 目标期刊

输出：

- 中文草稿
- 英文终稿
- 贡献表述版本
- 讨论段落建议

## 3. 智能体职责划分

建议后端采用多智能体协同，但前台仍保持单一产品体验。

### 3.1 论文理解智能体

负责：

- 解析论文结构
- 提取研究问题、理论、方法、发现、贡献

### 3.2 学术翻译智能体

负责：

- 中译标题
- 摘要中文重写
- 正文分段翻译
- 中英对照生成

### 3.3 术语与变量智能体

负责：

- 管理 glossary
- 生成变量翻译表
- 维持术语一致性

### 3.4 写作智能体

负责：

- 生成综述段落
- 生成结果解释
- 生成中英文写作文本

### 3.5 个性化记忆智能体

负责：

- 学习用户术语偏好
- 项目级术语沉淀
- 用户常用研究主题建模

## 4. 工作流编排逻辑

```text
导入论文
  -> 结构解析
  -> 学术理解
  -> 术语标准化
  -> 中文重写 / 对照生成
  -> 进入科研工作流
  -> 用户修正反馈
  -> 个性化记忆更新
```

## 5. 输出风格要求

### 5.1 中文文献综述风格

输出应接近可直接写入论文的中文段落，例如：

“现有研究主要从社会认知、角色互动和身份动态形成三个视角解释员工反馈行为。然而，对于反馈寻求如何在持续互动中塑造个体身份认同，现有文献仍缺乏系统讨论。”

### 5.2 中文结果解释风格

输出应偏管理学实证论文语感，而不是口语化总结。

### 5.3 英文投稿风格

应更加简洁、逻辑化，贴近 JAP、AMJ、ASQ 等期刊表述习惯。

## 6. 为什么这一层重要

如果系统只有翻译能力，用户看完论文后仍然要回到其他工具中整理思路。工作流层的价值在于把“阅读理解”转化为“研究产出”，让系统真正嵌入学者日常工作。
