# 翻译引擎与系统架构

## 1. 设计目标

学术翻译引擎的目标不是直译，而是生成可用于理解、写作和引用的中文学术表达，并在整个系统中保持术语与变量的一致性。

## 2. 核心原则

- 保留学术含义与理论精度
- 适配中文学术写作习惯
- 保持术语一致
- 支持中英对照
- 支持用户偏好记忆

## 3. 系统分层

```text
输入层
  -> 文档解析层
  -> 学术语义理解层
  -> 术语与变量标准化层
  -> 中文学术重写层
  -> 双语对照生成层
  -> 用户记忆与反馈层
```

## 4. 各层说明

### 4.1 输入层

支持：

- PDF 上传
- DOI 导入
- URL 导入
- 外部数据库同步

### 4.2 文档解析层

职责：

- 论文正文抽取
- 章节结构识别
- 图表标题识别
- 摘要与讨论部分识别
- 引用和参考文献分离

输出结果：

- 段落级文本
- 结构化章节树
- 元数据对象

### 4.3 学术语义理解层

职责：

- 提取研究问题
- 识别理论基础
- 判断研究方法
- 提取核心变量
- 提取发现和贡献

输出对象建议：

```json
{
  "research_question": [],
  "theories": [],
  "method": {},
  "variables": [],
  "findings": [],
  "contributions": []
}
```

### 4.4 术语与变量标准化层

职责：

- 建立统一术语映射
- 消除同一术语的多译
- 约束核心变量翻译
- 优先采用用户偏好

输入来源：

- 学科术语库
- 用户偏好词表
- 项目词表
- 当前论文上下文

### 4.5 中文学术重写层

职责：

- 标题学术化翻译
- 摘要重写
- 讨论与贡献中文化表达
- 结果解释重组

该层应输出符合中文学术写作风格的完整内容，而不是英文句式的机械映射。

### 4.6 双语对照生成层

职责：

- 生成中英双栏对照视图
- 支持段落映射
- 支持术语高亮
- 支持句子级引用定位

### 4.7 用户记忆与反馈层

职责：

- 存储术语偏好
- 记录用户纠正
- 跟踪项目级术语
- 为后续翻译提供约束

## 5. 翻译结果格式

每篇论文统一输出以下结构：

1. 标题翻译
2. 摘要重写
3. 核心内容提炼
4. 关键变量翻译表
5. 术语表
6. 可引用句

## 6. 质量控制机制

### 6.1 前置约束

翻译前自动加载：

- 用户偏好术语
- 管理学高频理论术语表
- 当前项目术语表
- 禁止误译列表

### 6.2 后置检查

翻译后进行：

- 术语一致性检查
- 变量名一致性检查
- 理论术语误译检查
- 中文可读性评分

### 6.3 人机协作

用户可在界面中：

- 修改单个术语译法
- 指定某构念标准表达
- 将本次译法保存为默认偏好

## 7. 学术翻译引擎的差异化

与普通翻译产品相比，本系统更强调：

- 先理解论文结构，再翻译
- 先统一变量与理论，再重写中文
- 输出可以直接用于中文写作与英文投稿准备

这也是系统成为“科研操作系统”而非“翻译工具”的基础。
