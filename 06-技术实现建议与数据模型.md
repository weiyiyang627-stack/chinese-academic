# 技术实现建议与数据模型

## 1. 总体技术架构

建议采用前后端分离加异步任务编排架构：

```text
Web / App 前端
  -> API Gateway
  -> Auth & User Service
  -> Paper Ingestion Service
  -> Translation Engine Service
  -> Workflow Service
  -> Terminology Memory Service
  -> Recommendation Service
  -> PostgreSQL / Redis / Vector DB / Object Storage
```

## 2. 前端建议

- Next.js
- TypeScript
- Tailwind CSS
- PDF.js
- i18n 中英切换框架

### 前端重点组件

- 中文论文卡片组件
- 中英双栏阅读器
- 术语 hover 卡片
- 变量详情抽屉
- 工作流输入面板
- 输出结果编辑器

## 3. 后端建议

- Python FastAPI 或 Node.js NestJS
- Redis 用于缓存与异步队列
- Celery 或 BullMQ 处理长文解析和翻译任务

### 核心服务

#### 3.1 Paper Ingestion Service

负责：

- 接收 PDF / DOI / URL
- 元数据抓取
- 文本抽取
- 结构化分段

#### 3.2 Translation Engine Service

负责：

- 标题翻译
- 摘要重写
- 全文翻译
- 双语对照生成
- 可引用句生成

#### 3.3 Semantic Extraction Service

负责：

- 提取研究问题
- 理论基础
- 方法
- 发现
- 贡献

#### 3.4 Terminology Memory Service

负责：

- 用户术语偏好
- 变量翻译表
- 项目级词库
- 一致性检查

#### 3.5 Workflow Service

负责：

- 文献综述工作流
- 变量构建工作流
- 实证分析工作流
- 写作工作流

#### 3.6 Recommendation Service

负责：

- 每日论文推荐
- 与研究主题的相关性打分
- 推荐用途标签生成

## 5. API 设计示例

### 5.1 导入论文

`POST /papers/ingest`

请求：

```json
{
  "source_type": "doi",
  "source_value": "10.xxxx/abcd"
}
```

响应：

```json
{
  "paper_id": "paper_123",
  "status": "processing"
}
```

### 5.2 生成翻译结果

`POST /papers/{id}/translate`

请求：

```json
{
  "mode": "full",
  "target_language": "zh-CN",
  "style": "academic"
}
```

### 5.3 提取论文核心结构

`POST /papers/{id}/extract`

响应字段建议：

- research_question
- theory
- method
- findings
- contribution
- variables
- glossary

### 5.4 调用工作流

`POST /workflow/lit-review`

`POST /workflow/construct-variable`

`POST /workflow/empirical-interpretation`

`POST /workflow/writing`

## 6. 数据模型建议

### 6.1 users

| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid | 用户 ID |
| locale | text | 默认语言 |
| research_interests | jsonb | 研究主题 |
| target_journals | jsonb | 目标期刊 |
| writing_preferences | jsonb | 写作偏好 |

### 6.2 papers

| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid | 论文 ID |
| title_en | text | 英文标题 |
| title_zh | text | 中文标题 |
| abstract_en | text | 英文摘要 |
| abstract_zh | text | 中文摘要重写版 |
| journal | text | 期刊 |
| authors | jsonb | 作者 |
| publication_status | text | 发表状态 |
| source_url | text | 原文链接 |
| parsed_content | jsonb | 结构化正文 |

### 6.3 terminology_entries

| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid | 术语 ID |
| term_en | text | 英文术语 |
| term_zh_standard | text | 标准中文译法 |
| term_zh_preferred | text | 用户偏好译法 |
| discipline | text | 学科 |
| notes | text | 说明 |

### 6.4 variable_entries

| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid | 变量 ID |
| name_en | text | 英文变量名 |
| name_zh | text | 中文变量名 |
| definition | text | 理论定义 |
| measures | jsonb | 测量方式 |
| proxies | jsonb | 替代变量 |
| related_terms | jsonb | 相关术语 |

### 6.5 translation_jobs

| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid | 任务 ID |
| paper_id | uuid | 论文 ID |
| mode | text | summary / full / bilingual |
| status | text | pending / running / done / failed |
| glossary_snapshot | jsonb | 当次术语快照 |
| output_payload | jsonb | 输出结果 |

### 6.6 workflow_runs

| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid | 运行 ID |
| user_id | uuid | 用户 ID |
| workflow_type | text | 工作流类型 |
| input_payload | jsonb | 输入 |
| output_payload | jsonb | 输出 |
| created_at | timestamptz | 创建时间 |

## 7. Prompt 与模型编排建议

建议为不同任务建立独立 prompt 模板：

- 论文结构抽取 prompt
- 学术中文摘要重写 prompt
- 术语一致性约束 prompt
- 变量表生成 prompt
- 可引用句生成 prompt
- 文献综述工作流 prompt

关键做法：

- 所有生成任务都先加载术语快照
- 长文翻译按章节分块，但在全局做术语统一
- 中文输出优先遵循学术写作风格而不是日常语言
